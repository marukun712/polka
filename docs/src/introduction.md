# はじめに

**polka**は、タグとリンクでつながる新しい分散型Webプロトコルです。「**検証可能なWeb 1.0**」をコンセプトに、個人サイトの自由さと、分散型技術による信頼性を両立させます。

## 開発の背景：既存Webの課題

現代のWebでは、サービス管理者がデータを保持し、APIを通じて提供しています。しかし、この構造には以下の課題があります。

**中央集権的なコントロール**

運営企業の判断により、アカウントがBANされたり、投稿へのアクセスが遮断されたりします。自分のデータであっても、完全にはコントロールできません。

**プラットフォーム依存**

API制限やサービス終了により、これまでの活動やコミュニティが失われるリスクがあります。プラットフォームに依存している限り、この不安は消えません。

## polkaの3要素

polkaは以下の3つの柱によって、サービスが消えてもデータが残るWebを実現します。

**1. データとサービスの分離**

ユーザー自身がサーバーでデータを公開し、アプリがそこに直接アクセスします。データはあなたのものです。

**2. 電子署名による検証可能性**

すべてのデータは暗号学的に署名され、改ざんを検出できます。誰が書いたか、改ざんされていないかが、数学的に保証されます。

**3. タグによるリンクのつながり**

タグを用いたグラフ構造により、中央管理者なしでコミュニティが自然発生します。興味のあるタグをフォローするだけで、つながりが生まれます。

## polkaでできること

polkaを使うと、以下のようなことが可能になります。

**静的ホスティング**

GitHub PagesやNetlifyなどの静的ホスティングで、自分のデータを公開できます。特別なサーバーソフトウェアは不要です。

**タグを通じた緩やかなつながり**

興味のあるトピックを持つユーザーを発見し、そのタグの投稿だけをフォローできます。ユーザー全体ではなく、特定の話題だけに興味を持つことができます。

### ユースケース例

**趣味のコミュニティ**

高橋ポルカが好きな人は、`anime/ikizulive/polka` タグをフォローします。このタグを使う人々が自然につながり、緩やかなコミュニティが形成されます。

**ブックマーク**

興味のあるリンクをpolkaで公開し、タグで分類します。同じタグを使う人々のブックマークをフォローすることで、情報を共有できます。

## polkaでできないこと

**リアルタイムチャット**

polkaは静的ファイルベースなので、リアルタイム性が求められる用途には適していません。

**大規模メディアの配信**

画像や動画は各自でホスティングする必要があります。polkaはメディアの配信インフラを提供しません。

**プライベートな投稿**

polkaのデータは基本的に公開されます。プライベートなやり取りには暗号化などの追加の仕組みが必要です。

## 技術スタック

polkaは、既存の技術を組み合わせて実現されています。

**ATProtocol (データ構造)**
Blueskyでも使われているATProtocolのデータ構造を部分的に採用しています。これにより、データ全体を効率的に検証できます。

**Nostr (ユーザー発見)**
既存のNostrインフラを活用し、タグを持つユーザーを発見します。Bloom Filterを使うことで、リレーに直接タグ情報を送信せず、効率的にマッチングします。

**did:web (アイデンティティ)**
ドメイン名と公開鍵を紐付けるために、did:webを使用します。既存のHTTPS/DNSを信頼の基盤とします。

**Git + 静的ホスティング (配信)**
データはGitで管理され、静的ファイルとして配信されます。GitHub Pages、Netlify、Caddyなど、どのホスティングでも動作します。

これらの技術を組み合わせることで、最小限のインフラで分散型Webを実現しています。
