# What is polka?

polkaは、自分のデータを完全に所有するための分散型SNS基盤です。

## polkaのコンセプト

近年、様々な分散型SNSが登場していますが、その多くが専用のリポジトリサーバーを起動したり、なんらかのサーバーに所属する必要がありました。

また、投稿をサーバーに集約しないNostrのようなモデルの分散型SNSの場合、データが不用意に拡散してしまい、データの所有権があいまいになってしまうという問題があります。

そこで、polkaは、「自分のPCでファイルを公開する」というWebの基本に立ち返り、ポータブルな静的ファイルを活用した、外部インフラへの依存を自分で選択できる分散型SNSの形を提案します。

### 静的ファイルベース
polkaにおけるアイデンティティ、コンテンツはすべて静的ファイルです。

つまり、ユーザーが自宅サーバーやGitHub Pages、またはIPFSなどの任意の場所でホスティングすることができます。

polkaがホスティング場所を縛ることはありません。ファイルを公開できる環境であればどこでもホスティングすることができます。

### ポータブルなリポジトリ
polkaにおけるユーザーデータは、ATProtocolの仕様に基づく単一のCAR(Content Addressable aRchives)ファイルです。

ATProtocolのデータ構造は、Merkle Treeの変種(Merkle Search Tree)で管理されており、ルートハッシュを比較することでデータの完全性を証明することができます。
さらに、このルートハッシュに対して電子署名を行うことで、単一ファイルでデータセット全体を署名することができます。

データをほかの場所でホスティングしたいときも、CARファイルをほかの場所に移動してDID DocumentのURLを更新するだけです。

### ドメインがアイデンティティ
polkaでは、did:webというドメインをベースとしたアイデンティティを使用します。
DID Documentという静的ファイルで、リポジトリを検証するための公開鍵とリポジトリへのファイルURLを指定することができます。

すべてのファイルが静的ファイルであるため、特別なリポジトリサーバーは必要ありません。
お使いのファイルサーバーやWebサーバーにそのまま統合することができます。

## polkaが目指すもの
- 自分のデータを完全に所有すること
- ドメインをベースとしたゆるいつながり

## polkaが目指さないもの
- 世界中のトレンドがわかるグローバルなSNS
- 高速な検索・集計
