# What is polka?

polkaは、ATProtocolをベースとした、検証可能なデータホスティングを提供するソフトウェアです。

## polkaのコンセプト

ATProtocolは、Blueskyのように多くのユーザーのデータを集約し、配信することを前提に設計されています。
そのため、PDSやリレーなど、常時稼働するサーバーが重要な役割を果たします。

polkaは、ATProtocolのデータ構造や検証可能性を活かしつつ、
大規模な集約やインデクシングを前提としない使い方を想定しています。

polkaでは、リポジトリを静的なファイルとして展開し、
HTTP経由で必要なデータだけを取得できる形で配信します。
データ自体は、通常のファイルサーバーやオブジェクトストレージ上に配置できます。

一方で、更新や署名のために、最小限の常時稼働するプロセスは必要になります。
polkaでは、その役割をできるだけ小さく保ち、
従来のPDSのような常時接続・常時配信を前提としない構成を取ります。

このようにpolkaは、ATProtocolを
個人のデータを扱うための、より軽量なホスティング用途にも使える形で提供します。

### 静的な資産としてのデータ配信

polkaは、ATProtoの堅牢なデータ構造(Repository/MST)を継承しつつ、それを「静的な資産」として配信することに特化したソフトウェアです。

従来のCARファイルによる配信では、データの断片が必要な際もファイル全体をダウンロードする必要がありました。polkaは、CARファイルをベースとしつつも、リポジトリをブロック単位の静的ファイルに展開して配信することで、HTTP経由で必要なブロックだけをオンデマンドに取得することを可能にします。

### 自己主権的なアイデンティティ

あなたのアイデンティティは、DIDによって定義されます。あなたの秘密鍵で署名されたデータは、どのサーバーに置かれていようとも、それが「ドメインと紐づけられた公開鍵の持ち主のもの」であることを証明できます。

### 目指すもの

Googleに身元を預け、Twitterにデータを預ける。そんな現在のインターネットの構造を、polkaは「自己署名された検証可能なデータ」という新しい基盤へと作り変えます。
