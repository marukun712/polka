# データ構造

polkaは、グラフ、プロフィール、投稿などの多様なデータを効率的に検証するため、分散型SNS技術を応用したデータ構造を採用しています。

## Merkle Search Tree
投稿データだけでなく、タグの状態(グラフ構造)をまとめて検証するためにMSTを使用します。

*   **改ざん検知**: 複数のデータをまとめてハッシュ化し、そのルートハッシュに電子署名を行います。これにより、グラフ全体が正しい状態であることを効率よく証明できます。
*   **高速検索**: ソートされた構造により、特定のノードを迅速に検索可能です 。

## BlockStore
物理的なデータの保存と同期には、**BlockStore**形式を利用します。

*   **Key-Value構造**: データのハッシュ値を**Key**、実データを**Value**として管理します。
*   **部分同期の効率化**: クライアントはリポジトリ全体をダウンロードする必要はなく、必要なハッシュのみをWebサーバーから取得すれば済みます。
