# アイデンティティと検証

polkaでは、特定のサービスアカウントではなく、**公開鍵**がユーザーのアイデンティティとなります。

## did:webによる紐付け
ドメイン名と公開鍵を紐付ける仕組みとして、**did:web**を採用しています。

*   **信頼の基盤**: ドメインの真正性自体は、既存のHTTPS/DNSインフラに依存します。
*   **検証プロセス**: クライアントはデータを取得する際、`did:web`から公開鍵を取得し、付与された電子署名を検証します。

## ドメイン変更時の同一性証明
もしドメインが変更された場合でも、新しいドメインに以前と同じ公開鍵を配置することで、「公開鍵が同一 ＝ 同一人物」であることを継続的に証明可能です。

## 信頼の積み重ね
さらに、暗号学的な証明に加え、過去の発信の積み重ねそのものが信頼の根拠となると考えます。
